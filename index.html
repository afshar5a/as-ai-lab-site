Core definitions
Cost per Effective Request (CPER)
CPER = (TokenCost + Overhead) / EffectiveRequests
Effective Requests
EffectiveRequests = TotalRequests × (1 − CacheHitRate)
Break-even Revenue per User
RPU_be = MonthlyCost / MAU
Working principle
Margin is an architectural property. If architecture ignores economics, growth amplifies inefficiency.

Framework

A modeling approach that treats AI systems as economic systems — linking architectural choices to cost, break-even thresholds, and margin resilience under scale.

1) Architecture → Cost Map

Translate design choices into cost drivers: token budgets, retrieval depth, tool calls, caching, and orchestration.

Input/Output tokens Overhead per request Cache economics

2) Margin Resilience

Stress-test unit economics under growth and volatility: MAU growth, usage intensity, pricing shifts, and prompt drift.

Sensitivity analysis Break-even thresholds Risk levels

3) Optimization Levers

Identify high-leverage interventions: compression, retrieval tuning, caching strategy, token caps, and model routing.
